# Solvara Safaris

An ultra-luxury safari concierge web application targeting ultra-high net worth 
individuals, family offices, and personal managers. Built as a university web 
development group project.

**Live site:** https://solvara-xi.vercel.app  
**GitHub:** https://github.com/mahakidnani/solvara

---

## Project Overview

Solvara Safaris is not a standard travel booking site. It is designed as a 
private digital estate — the online equivalent of a bespoke concierge service. 
The site covers twelve African safari destinations and provides end-to-end 
journey curation from doorstep collection to post-trip memory book delivery.

The project began as an earlier prototype called WildVerse (built by a team 
member), which had strong technical foundations but no defined target audience 
or brand direction. After structured design thinking sessions and competitor 
analysis — referencing Singita, Abercrombie & Kent, and &Beyond — the team 
pivoted to a fully redefined concept with a clear UHNWI audience, dark 
editorial aesthetic, and the Solvara brand identity.

## Getting Started
```bash
# Clone the repository
git clone https://github.com/mahakidnani/solvara.git

# Navigate into the project
cd solvara

# Install dependencies
npm install

# Start the development server
npm run dev

# Build for production
npm run build
```

Open `http://localhost:5173` in your browser.

## Key Design Decisions

**Why dark aesthetic?** Competitor analysis showed every major safari brand 
(Singita, &Beyond, Abercrombie & Kent) used light backgrounds with warm earthy 
tones. A fully dark editorial aesthetic was an open competitive space and 
positioned Solvara closer to luxury hospitality than outdoor adventure travel.

**Why no prices?** Usability interviews revealed that price listings made sites 
feel like commodities. All enquiries route through the Concierge page to 
maintain a bespoke, consultative feel.

**Why React?** Written approval from the course lecturer (Dr. Brent Kelly) was 
obtained before development began, confirming React.js was acceptable given the 
group's prior familiarity with the framework.

**What didn't make it?** The interactive 3D globe, Virtual Verse immersive 
audio environments, and full membership portal were scoped out due to time 
constraints. The concierge form serves as the primary booking flow in their 
place.
